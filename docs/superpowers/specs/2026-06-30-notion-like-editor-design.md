# Notion-like Independent Editor Design

## Overview
Every card (todo, note, inbox item) can be clicked to open a dedicated, Notion-like rich text editor. This allows users to record extensive thought context without cluttering the main dashboard.

## UI/UX
- **Trigger**: Clicking the card body or text opens the editor. For Todo items, clicking the checkbox ONLY toggles completion, while clicking elsewhere opens the editor.
- **Layout Modes**: The editor opens as a large modal by default (with a blurred backdrop). A toggle button in the top right allows switching between Modal, Right Side Panel, and Full-Screen modes.
- **Title Sync**: The top of the editor displays a large, editable title field synced with the card's `text` property. Changing the title inside the editor immediately updates the card text on the dashboard.

## Editor Architecture
- **Core Engine**: `Editor.js` loaded via CDN to provide a true block-based editing experience.
- **Plugins/Tools**:
  - Header, List, Checklist, Quote, InlineCode, Marker.
- **Interactions**:
  - Floating inline toolbar (Bold, Italic, Highlight) when text is selected.
  - Markdown shortcuts support (e.g., `#` for headers) provided by Editor.js native or plugin functionality.

## Data Flow & Storage
- **Lazy Loading**: Note content is NOT loaded with the initial dashboard payload. It is fetched from Firestore only when a specific card is clicked.
- **Storage Path**: Stored in a subcollection `details` under the specific card's document, in a document named `note`.
  - Path: `artifacts/{appId}/users/{userId}/{categoryId}/{cardId}/details/note`
  - Field: `blocks` (Array of JSON blocks generated by Editor.js), `time` (timestamp), `version` (string).
- **Auto-save**: Editor listens to `onChange` events. After 1000ms of inactivity (Debounce), the content is automatically synced to Firestore.

## Implementation Phases
This is scoped sufficiently for a single implementation plan:
1. Setup UI wrapper (Modal/Sidebar/Fullscreen layout) and update card click logic.
2. Integrate Editor.js and plugins via CDN.
3. Implement lazy loading, auto-save to Firestore, and Title syncing logic.
